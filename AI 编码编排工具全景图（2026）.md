# AI 编码编排工具全景图（2026）

您提到的 OpenSpec 和 Superpowers 属于一个新兴但快速演进的领域：**AI 编码编排（AI Coding Orchestration）**。它们不是简单的"AI 写代码工具"，而是让 AI 按规范、按流程、可重复地完成复杂任务的方法论与工具集。

下面按"层级"系统梳理这个领域。

---

## 一、先理解：什么是"AI 编码编排"

| 单纯的 AI 助手 | AI 编码编排 |
|--------------|------------|
| "帮我写个登录页" | "按这套规范写 → 自检 → 审查 → 测试 → 提交" |
| 一次性、不可控 | 流程化、可重复、可审查 |

**核心解决三个问题：**
1. **质量稳定性**：避免每次结果差异巨大
2. **复杂任务拆解**：让 AI 处理需要多步推理的任务
3. **团队协作一致性**：让团队不同人用 AI 得到一致输出

**按层次分为 5 类工具：**

```
┌────────────────────────────────────────────┐
│ Layer 5: 多 Agent 协作框架                  │  CrewAI, LangGraph, AutoGen
├────────────────────────────────────────────┤
│ Layer 4: 自主 Agent（云端/异步）            │  Devin, Cursor BG Agent, Factory
├────────────────────────────────────────────┤
│ Layer 3: 编辑器内 AI 编排                   │  Cursor, Claude Code, Cline
├────────────────────────────────────────────┤
│ Layer 2: 工作流/技能/规范系统               │  Superpowers, Claude Skills, MCP
├────────────────────────────────────────────┤
│ Layer 1: Spec 驱动开发方法                  │  OpenSpec, Spec Kit, Kiro, BMad
└────────────────────────────────────────────┘
```

---

## 二、Layer 1：Spec 驱动开发

**核心理念**：先写"规格说明（Spec）"，AI 按 Spec 实现，避免漫无目的的对话式编程。

| 工具 | 出品方 | 特点 | 适用场景 |
|------|--------|------|----------|
| OpenSpec | 开源社区 | 轻量、Markdown 驱动、规范变更也用 PR 流程 | 中小项目、想引入规范但不想太重 |
| GitHub Spec Kit | GitHub | 官方出品、与 Copilot/Codex 深度集成 | GitHub 生态团队 |
| Kiro | AWS | 全功能 IDE、规范+实现+测试一体 | 想要"全套方案"的团队 |
| BMad-Method | 开源 | 角色化（PM、架构师、开发等多 agent） | 大型项目、流程严谨的团队 |
| Tessl | Tessl | 强调"AI Native 开发"、规范即代码 | 早期产品，关注度高 |

**共性流程：**
```
需求 → 写 Spec → AI 生成实现 → 跑测试 → 审查 → 合并
        ↑___________↓
       Spec 变更也走 PR
```

**适合你：**
- 副业 / 独立项目 → **OpenSpec**（最轻）
- 公司团队推广 → **Spec Kit** 或 **BMad-Method**
- 想体验最先进 → **Kiro**

## 三、Layer 2：工作流/技能系统

**核心理念**：把"做某件事的最佳实践"打包成可复用的"技能（Skill）"，AI 在合适时机自动调用。

### 主流方案

| 工具 | 形态 | 关键能力 |
|------|------|----------|
| Superpowers | 开源 Skills 集（cursor-public/superpowers） | 调试、TDD、计划编写、代码审查、并行 agent 调度等 30+ skills |
| Claude Skills（Anthropic） | Claude/Claude Code 原生 | 官方 SDK 支持、可挂载到任意 Claude 应用 |
| Cursor Skills | Cursor 内置 | 项目级 skill、可自动触发 |
| Cursor Hooks | Cursor 内置 | 在 agent 事件点触发脚本（如 commit 前自检） |
| Cursor Rules / AGENTS.md | Cursor 内置 | 项目编码规范、AI 自动遵守 |
| MCP（Model Context Protocol） | Anthropic 推出，已成事实标准 | 工具/资源/Prompt 三件套，让 AI 安全接外部能力 |
| Continue.dev Workflows | 开源 | 自定义工作流、本地运行 |

### Skills vs Rules vs MCP 的区别

| 类型 | 描述 | 特点 |
|------|------|------|
| Rules | "永远要这么写" | 编码规范（被动遵守） |
| Skills | "遇到 X 时这么做" | 流程方法（主动调用） |
| MCP Server | "我能做 Y" | 外部能力（工具接入） |
| Hooks | "Y 发生时跑 Z" | 自动化触发（事件驱动） |

**适合你：**
- 个人项目 → **Cursor Rules + Superpowers Skills**（你已有 Skills 文件夹）
- 团队推广 → 自建 **MCP Server** + 共享 Skills 库
- 接入企业系统 → **MCP**（接 Jira、内部 API、Datadog 等）

## 四、Layer 3：编辑器内 AI 编排（IDE 类）

**核心理念**：把编排能力做进 IDE，让用户在写代码现场获得 AI 协作。

| 工具 | 定位 | 特点 |
|------|------|------|
| Cursor | AI-first IDE | 编排能力最完整：Agent 模式、Plan 模式、Background Agent、Skills、Rules、Hooks、MCP |
| Claude Code | Anthropic 终端工具 | 原生 Skills、Plugins、Hooks，CLI 玩家最爱 |
| Cline / Roo Code | 开源 VS Code 插件 | 自带 plan/act 模式、可挂任意模型 |
| Aider | 开源命令行工具 | Git 友好、轻量、CLI Power User 首选 |
| Continue.dev | 开源 VS Code/JetBrains 插件 | 高度可定制、企业自部署 |
| Windsurf（Codeium） | AI IDE | "Cascade" 流模式，竞品 Cursor |
| GitHub Copilot Workspace | GitHub 官方 | Issue → 计划 → 实现，与 GitHub 流程深度整合 |

### 关键差异：
- **Cursor / Windsurf**：完整 IDE 体验，新手友好
- **Claude Code / Aider**：CLI / 终端，适合资深开发
- **Cline / Continue**：开源、可自定义、企业可控

**适合你：** 你已经在用 Cursor，继续深用就够了，重点掌握它的 **Skills + Rules + MCP**。

---

## 五、Layer 4：自主 Agent（异步/云端）

**核心理念**：交给 AI 一个任务，它自己规划、执行、提交 PR，人不在线也能干活。

| 工具 | 出品方 | 特点 |
|------|--------|------|
| Devin | Cognition | 最早的"AI 软件工程师"产品，能跑长任务 |
| Cursor Background Agent | Cursor | 云端异步 agent、可并发、Slack/GitHub 触发 |
| Cursor Bugbot | Cursor | 自动 PR Review |
| Factory.ai | Factory | 企业级 AI 工程师平台 |
| Tessl | Tessl | Spec 驱动 + 自主 agent |
| Codegen | Codegen | 自主 PR 生成 |
| GitHub Copilot Agent | GitHub | Issue 自动转 PR |
| Replit Agent | Replit | 全栈应用一键生成 |
| Bolt.new / v0 / Lovable | StackBlitz / Vercel / Lovable | Web 应用生成（更偏前端原型） |

**典型用法：**
```
GitHub Issue 提一个 Bug → 触发 Background Agent → 30 分钟后收到 PR
```

**适合你：**
- 个人开发者：**Cursor Background Agent**（你已有账号）
- 想体验顶级：**Devin**（贵）
- 团队场景：**Factory.ai**

## 六、Layer 5：多 Agent 协作框架（程序员搭系统）

**核心理念**：你自己编排多个 AI agent 协作，做应用/系统/工作流。

| 框架 | 语言 | 特点 |
|------|------|------|
| LangGraph | Python/TS | LangChain 出品、状态机模型、最主流 |
| CrewAI | Python | 角色分工（PM/Dev/QA），易上手 |
| AutoGen | Python | 微软出品，多 agent 对话式协作 |
| OpenAI Swarm | Python | OpenAI 实验性框架，轻量 |
| Mastra | TypeScript | TS 生态最完整的 agent 框架 |
| Inngest Agent Kit | TypeScript | 集成 Inngest 工作流 |
| VoltAgent | TypeScript | 新兴，开发者体验好 |
| LlamaIndex Workflows | Python | 数据/RAG 场景强 |
| Vercel AI SDK | TypeScript | 前端友好、轻量 agent 能力 |

**典型用法：** 构建你自己的 AI 应用（客服机器人、自动数据分析、AI HR 助手等）

**适合你：**
- TS 全栈方向 → **Mastra/Vercel AI SDK/Inngest Agent Kit**
- Python 方向 → **LangGraph/CrewAI**

## 八、选型决策树

```
你想解决什么问题？
│
├─ "让 AI 写代码更靠谱"
│  → IDE（Cursor/Claude Code） + Rules + Skills
│
├─ "让 AI 按团队规范工作"
│  → Spec Kit / OpenSpec + AGENTS.md + 自建 Skills
│
├─ "让 AI 异步替我干活"
│  → Cursor Background Agent / Devin / Factory
│
├─ "让 AI 接入公司内部工具"
│  → MCP Server（自建或用现成的）
│
└─ "我要构建 AI 产品"
   → 多 Agent 框架（Mastra/LangGraph/CrewAI）
```

## 九、值得关注的趋势（2026）

1. **MCP 已成事实标准**：Anthropic 提的 Model Context Protocol，OpenAI、Google 都已支持，学一次到处用
2. **Skills 模式赢了 Plugins 模式**：Markdown + 元信息描述的 Skill 比硬编码 plugin 更灵活
3. **Spec 驱动正在主流化**：OpenSpec / Spec Kit / Kiro 都在卷这个方向
4. **Background Agent 成为标配**：异步、并发、长任务执行
5. **AGENTS.md 成新规范**：类似 README，但写给 AI 看的项目说明
6. **Hooks / 事件驱动**：Cursor、Claude Code 都加了 hooks，自动化能力变强
7. **多模型路由器**：Vercel AI SDK、Continue 等让你按任务选模型
